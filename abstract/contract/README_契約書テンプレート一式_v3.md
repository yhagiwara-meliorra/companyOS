## いまの段階で特に埋めるべき項目
- [ ] 会社名、住所、代表者名
- [ ] 個人情報取扱責任者、インシデント統括責任者
- [ ] 使う外部サービス（Vercel / Supabase / OpenAI / Anthropic / その他）
- [ ] AIログ保存先と保持期間
- [ ] 東京管轄でよいか
- [ ] 料金・責任上限・検収期間を案件に合わせる
- [ ] 顧客データ削除期限、再委託の有無、学習利用設定
- [ ] 開示等請求の受付窓口、本人確認方法、手数料方針
- [ ] AI SaaS利用規約のサービス別紙
- [ ] AIリーガル運用フローの最終承認者

## 運用上の注意
- [ ] フォーム公開前にプライバシーポリシー、利用規約、開示等請求対応手順の整合を確認する
- [ ] 外部AIサービスを使う前に委託先管理台帳へ登録する
- [ ] 10万円以上の費用変動可能性がある変更は人間承認とAIログ記録を必須にする
- [ ] CEO AI の設計変更は変更管理台帳へ必ず記録する
- [ ] 新しいサービスを増やすたびに、CEO から AI Legal へ改定依頼票を出す
- [ ] 実締結前・公開前に弁護士レビューを入れる

## 新サービス追加時の最短フロー
1. CEO が `13_AIリーガル改定依頼テンプレート_日本語雛形.md` を記入
2. AI Legal が影響分析と改定対象文書を特定
3. AI CEO が憲法整合性レビュー
4. Product / Eng が実装差分を確認
5. AI Legal が `10_AI_SaaS利用規約_日本語雛形.md` のサービス別紙と関連文書を赤入れ
6. 人間が最終承認
7. README / 台帳 / 公開文書を更新
