# AI CEO Dialogue Template

人間が AI CEO と対話するときに使う運用テンプレート。

## 1. Thread Setup

### Thread Type

- `company_strategy`
- `new_product`
- `product_revision`
- `service_addition`
- `go_to_market`
- `legal_policy_change`
- `incident_review`
- `operating_review`

### Thread Metadata

- スレッドID:
- 作成日:
- 起票者:
- Thread Type:
- 緊急度: 通常 / 高 / 緊急
- 期待アウトプット: 検討 / 方針決定 / 実装着手 / 公開判定 / 振り返り

## 2. 事前入力

### 必須入力

- 今回議論したいテーマ
- 背景
- なぜ今それを考える必要があるか
- 対象顧客
- 解決したい課題
- 現状の代替手段
- 今ある仮説
- 今回決めたいこと

### 任意入力（あると精度向上）

- 対象市場の規模
- 既存顧客の声
- 競合一覧
- 現在のKPI
- 既存プロダクトとの関係
- 既存の法務・契約・ポリシーへの影響
- 想定コスト増減
- 使用予定の AI / 外部サービス

## 3. 憲法整合性ゲート

1. Mission に整合するか
2. Vision に整合するか
3. Solve に整合するか
4. 次世代に説明できるか
5. 外部不経済を減らすか
6. 透明性があるか
7. 個人を強くするか
8. スケール可能性があるか
9. 禁止領域に該当しないか

## 4. フェーズ別対話

### 4.1 課題定義

- 誰が最も困っているのか
- 何にどれくらい困っているのか
- その痛みは頻度が高いか
- いま何で代替しているのか
- その代替手段はなぜ不十分か
- その課題は本質的か、表面的か
- 本当に支払いが発生するか
- 緊急性があるか

### 4.2 世界観・意義

- この課題を解くと、どんな世界が実現するか
- それは Vision にどうつながるか
- それは個人を強くするか
- 次世代にとって正当化できるか
- 単なる効率化ではなく、文明的価値を持つか

### 4.3 ソリューション仮説

- 解決案は妥当か
- AI を使う必然性はあるか
- AI でなくてもよい部分は何か
- 人間が担うべき意思決定は何か
- 倫理的・社会的な懸念は何か
- 利用者を弱くしないか
- 不透明なブラックボックス化を招かないか

### 4.4 ビジネスモデル・経済性

- 誰がお金を払うのか
- 何に対して払うのか
- 単発課金か継続課金か
- どの指標が value metric になるか
- 市場規模は ARR 100億円級を狙えるか
- CAC / 運用コスト / 粗利に無理がないか
- 資本がある人が払い、資本がない人にも価値が届く設計にできるか

### 4.5 MVP 定義

- 最小限で価値を出すには何が必要か
- 最初に削るべき機能は何か
- 最初の顧客は誰にすべきか
- どの成果物が出れば使われるか
- 成功条件は何か

### 4.6 実装・運用方針

- Claude / GPT / LangGraph / Supabase / Next.js の役割分担
- 保持すべきデータ
- human approval の挿入位置
- 残すべきログ
- node / graph 分割
- 先行実装と後回しの切り分け

### 4.7 Go-to-Market

- 最初の顧客獲得はどう進めるべきか
- どの業界 / 職種から入るべきか
- 価値提案は何か
- 価格の考え方はどうするか
- 営業以外の導線（紹介・PoC・コミュニティ）が有効か
- 最初の説明文は何か

### 4.8 法務・データ・ポリシー確認

- 個人情報を扱うか
- 外部 AI サービスに顧客データを投入するか
- 新しいサービス追加に該当するか
- 規約 / ポリシー / 内部規程の更新が必要か
- 開示等請求対応、漏えい対応、委託先台帳に影響があるか
- AIリーガル起票が必要か

### 4.9 リスク確認

- 最大の失敗要因は何か
- 法務・倫理・運用上の懸念は何か
- 市場から受け入れられない可能性は何か
- AI が誤ると致命的になるポイントは何か
- どの論点は必ず人間が判断すべきか
- 実装前に先に検証すべきことは何か

## 5. 人間承認ゲート

以下のいずれかに該当する場合は人間承認必須。

1. 支払い金額が10万円以上変わる可能性のある変更
2. CEO AI の設計変更
3. 外部公開文書の大幅改訂
4. 個人情報・要配慮情報・機密情報の新規取扱い
5. 大きなブランド毀損、炎上、規制対応の可能性
6. 顧客への重要な対外約束
7. 倫理的に境界が曖昧な案件

## 6. 意思決定フェーズ

- Go / Hold / Reject
- なぜその結論か
- reversible / irreversible
- 次に何を検証するか
- 誰がいつまでに何をするか
- 何をログに残すか

## 7. 最後に確認すること

- Decision と Next Action に必ず落とす
- 新サービス / 料金変更 / 新データ取得 / 新外部AI追加時は AIリーガル起票
- 1スレッド1テーマを原則にする
- 憲法整合性 / リスク / 実装可能性 / 収益性の4軸で評価する
