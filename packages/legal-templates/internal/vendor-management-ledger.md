# 委託先管理台帳（日本語雛形）

> 目的: 個人情報、顧客機密、AI処理対象データに触れ得る外部サービス、クラウド、委託先、再委託先候補を一覧化し、選定・見直し・終了管理を行うための台帳。

## 1. 登録対象
以下のいずれかに該当する先は原則登録する。
- 個人情報又は個人データにアクセスし得る先
- 顧客機密、契約情報、ソースコード、AIログにアクセスし得る先
- 外部AIサービス、埋め込み、OCR、翻訳、分析ツール等
- 開発、デザイン、営業支援、採用支援等の業務委託先
- ホスティング、データベース、ストレージ、監視、認証、メール配信等のSaaS

## 2. 登録・見直しルール
1. 新規導入前に仮登録し、用途、データ種別、契約条件、責任者を記入する。
2. 個人情報又は高機密データを扱う場合、導入前に責任者レビューを行う。
3. 少なくとも半年ごと、又は契約条件・サービス仕様・データフロー変更時に見直す。
4. 解約又は利用終了時は、データ削除、アカウント無効化、鍵の削除、台帳更新を行う。

## 4. 導入前レビュー項目
- [ ] 利用目的が明確
- [ ] 代替手段を確認済み
- [ ] 個人情報の要否を確認済み
- [ ] 顧客契約上、当該利用が許容される
- [ ] 学習利用の有無を確認済み
- [ ] 保存期間、リージョン、再委託の有無を確認済み
- [ ] 契約条項、責任制限、終了時削除方法を確認済み
- [ ] 事故時の連絡窓口を確認済み
- [ ] 管理者と見直し日を設定済み

## 5. 解約・利用終了時チェック
- [ ] アカウント停止
- [ ] APIキー / トークン失効
- [ ] データ削除依頼又は削除確認
- [ ] 社内接続解除
- [ ] 顧客向け再委託一覧の更新
- [ ] 台帳ステータス更新

## 6. 運用メモ
- 個人情報や高機密データに触れるベンダーは、特にレビュー履歴を残す
- 無料プランや個人契約の利用は原則避ける
- AIサービスは「学習利用」「ログ保持」「モデル改善利用」を別欄で明示すると事故を防ぎやすい
